## 1. Why / context
The final T6 piece. A multisig/miniscript wallet needs the OTHER cosigners' keys, which one seed can't produce ‚Äî so the user SUPPLIES a complete wallet-policy md1 (made elsewhere: the toolkit, another SH). The device derives the user's OWN key from the typed seed, CROSS-MATCHES it to one of the descriptor's @N slots, then engraves the supplied md1 VERBATIM + the user's mk1 (policy-bound stub) + ms1. **ZERO new encoder** (architect mechanism ii) ‚Äî the md1 is engraved as-is; the only net-new logic is the slot cross-match. The restore *primitives* (`expandedToDescriptor` + `address`) are already shipped and handle sortedmulti (the restore render-path *wiring* is net-new ‚Äî R0-I3 / R0-n2).

## 2. Scope

### IN
- **A new top-level `engraveMultisig` program** (between `engraveSingleSig` and `qaProgram`; 8-site lockstep + 3 nav-tests updated; `qaProgram` stays debug-only).
- **Gather the supplied md1** over NFC via the T5 `bundleGatherFlow` (its scan loop accumulates chunked md1), then a **NET-NEW `extractSuppliedMd1(cards) ([]string, ok)`** filter (R0-I1; do NOT reuse `singleSigReadbackCards` ‚Äî it *requires* BOTH an mk1 AND an md1, `gui/singlesig_verify.go:38`, the opposite of a multisig supply): exactly ONE `cardMD1` ‚Üí return its verbatim `[]string`; **0 md1 ‚Üí refuse** ("no wallet policy supplied"); **‚â•2 md1 ‚Üí refuse** ("supply exactly one wallet policy"); **any `cardMK1`/`cardMS1` present ‚Üí refuse** ("supply only the wallet-policy md1") ‚Äî NB (R0-n1): a `cardMS1` cannot arise from the gather path (ms1 is refused upstream at `classify`‚Üí`clsMs1Refuse`, `gui/bundle.go:64-71,130-131`), so the `cardMS1` clause is DEFENSIVE; only the `cardMK1` refusal is operator-reachable (a chunked mk1 over NFC). Then decode via `md.ExpandWalletPolicyChunks` ‚Üí `Template` + per-@N `[]ExpandedKey`.
- **Full-policy gate:** the supplied md1 MUST be a full wallet policy ‚Äî every slot `XpubPresent`. Template-only / any-slot-missing-xpub ‚Üí REFUSE ("the supplied descriptor has no public keys to match").
- **Typed-only seed** (`seedEntryFlow`; NEVER scan‚Üíderive) + optional passphrase.
- **Slot cross-match (`findUserSlot`, NET-NEW, D14 Critical):** for each `XpubPresent` slot, derive the user's account key from the seed AT that slot's `OriginPath` (`deriveAccountXpub`), decode to the canonical `(chainCode [32]byte, compressedPubkey [33]byte)` (`decodeXpubBytes` ‚Äî returns fixed arrays), and match iff **`bytes.Equal(cc[:], slot.Xpub[0:32]) && bytes.Equal(pk[:], slot.Xpub[32:65])`** ‚Äî the canonical-pair compare over the FULL 32+33 bytes, **NEVER base58** (the supplied xpub carries different parentFP/depth metadata) and **NEVER `==`** (R0-I2: `[N]byte == []byte` does not compile; `bytes.Equal` over slices, comparing all 33 pubkey bytes incl. the trailing byte of `[32:65]`). Exactly-one match ‚Üí the user's slot (record index + origin); ZERO ‚Üí REFUSE; **ambiguous (‚â•2 slots match ‚áí the SAME seed legitimately appears at two cosigner slots under DISTINCT origin paths)** ‚Üí engrave the FIRST-by-index slot's mk1 (deterministic; policy+stub identical across slots, only the mk1 Path differs) + a "key reused at slots @i,@j" notice.
- **Derive the user's leg:** mk1 = `mk.Encode(Card{Network: "mainnet", Path: matchedOrigin.String(), Fingerprint: hex(masterFP), Stubs: [][4]byte{WalletPolicyIDStubChunks(suppliedMd1)}, Xpub})` (the stub binds to the SUPPLIED policy; `Network` is a LABEL only per R0-m1 ‚Äî not serialized into the mk1 bytecode `mk/encode.go:69-95` nor compared by `bundle.Verify` ‚Äî set for consistency with the single-sig flagship `gui/singlesig_derive.go:74`); ms1 = `EncodeMS1(m.Entropy())` (full) ‚Äî gate `m.Valid()` first.
- **Engrave (full / watch-only):** full = supplied md1 (verbatim) + user mk1 + user ms1; watch-only = supplied md1 + user mk1 (skip ms1) + the ms1 reminder. Reuse `bundleEngrave`.
- **verify-bundle (user's slot only):** re-type seed ‚Üí re-cross-match ‚Üí re-derive the user's leg ‚Üí `bundle.Verify` (unchanged ‚Äî stub-binding + mk1 fp/xpub/path + md1-exact-string + ms1-entropy; watch-only both-empty-ms1 skip). Other slots are public-given/unverified-by-design.
- **restore doc (NET-NEW multisig render path, R0-I3 ‚Äî the shipped `restoreDocFlow` `gui/singlesig_restore.go:118` takes single-sig scalars and builds a single-sig descriptor internally; it CANNOT be reused):** call `expandedToDescriptor(tpl, keys)`; on `expandOK` (sortedmulti ‚Üí P2WSH/P2SH/P2SH_P2WSH) ‚Üí `address.Receive/Change` show the multisig addresses; on `expandTemplateOnly`/`expandUnsupported` (unsorted multi/multi_a/taptree/complex miniscript) ‚Üí display the descriptor read-only ("addresses unavailable") with **NO `address.*` call** (`expandedToDescriptor` returns a nil descriptor for non-sortedmulti `gui/md1_expand.go:36-49`, so a wrong-address verify is structurally impossible). Display-only, no secret.
- **Mainnet-only** (the cross-match is network-agnostic; derive on mainnet, matching T6a/#10b).

### OUT (deferred)
- On-device policy AUTHORING / the picker + `md.EncodeMultisig` (T6c ‚Äî out of scope, FOLLOWUPS). Self-multisig (constellation hard-rejected). Testnet (follow-on). The device NEVER constructs a multisig md1 ‚Äî it only engraves a supplied one verbatim.

## 4. Faithfulness / security spine
- **Seed typed-only, NEVER NFC** (the `gui/scan.go` footgun); ms1 engraved onto owner-held steel only, never NFC. Per-leg scrub (D11): gate `m.Valid()`, `defer wipeBytes` entropy, scrub the mnemonic after the last derive (the cross-match derives at each slot until matched ‚Äî scrub after); `deriveAccountXpub` scrubs seed/master/intermediates internally.
- **`.Neuter()`** ‚Äî no xprv ever serialized/engraved/NFC'd; restore-doc no secret.
- **D14 (the wrong-wallet guard):** the cross-match compares the CANONICAL chainCode‚Äñpubkey (not base58); REFUSE on zero matches ‚Äî never engrave a backup for a wallet the seed isn't a cosigner of.
- **Verbatim:** the supplied md1 is engraved as-is (the device never re-encodes a supplied descriptor ‚Äî it lacks an md multisig encoder by design).
- **Faithful-or-refuse:** address-verify/restore only for the bip380-expressible subset; non-bip380 shapes engrave verbatim but are display-only (never mis-verified against a wrong address).

## 5. Acceptance gate (TDD)
1. **Gather + cross-match (D14):** `extractSuppliedMd1` returns the single supplied md1 from a one-`cardMD1` gather; **0 md1 ‚Üí refuse, ‚â•2 md1 ‚Üí refuse, any `cardMK1`/`cardMS1` present ‚Üí refuse** (R0-I1). For a full-policy 2-of-3 `wsh(sortedmulti)` md1 where the user's seed is slot @1, `findUserSlot` returns index 1 + the slot's origin (`bytes.Equal` canonical-pair compare ‚Äî a base58 with different parentFP still matches); a seed that is NOT a cosigner ‚Üí zero match ‚Üí refuse; the SAME seed at two cosigner slots under DISTINCT origins (e.g. @0 and @2) ‚Üí first-by-index + notice (R0-m4). Template-only / any-slot-missing-xpub ‚Üí refuse.
2. **Derive the user's leg:** mk1 stub == `WalletPolicyIDStubChunks(suppliedMd1)` (bound, decoded-field assert not raw-string); mk1 Path == the matched slot origin; ms1 round-trips the entropy.
3. **Engrave:** full = 3 cards (ms1 + user mk1 + supplied md1 verbatim); watch-only = 2 + reminder; the supplied md1 strings are engraved UNMODIFIED.
4. **verify-bundle:** re-derive + `bundle.Verify` PASS for the user's slot; a mutated user mk1/md1/ms1 ‚Üí FAIL; watch-only (ms1-less) verify works.
5. **restore doc (new multisig render path, R0-I3):** a sortedmulti supplied md1 ‚Üí `expandedToDescriptor`‚Üí`address.Receive/Change` match the expected multisig address (golden); a non-bip380 miniscript md1 ‚Üí `expandedToDescriptor` returns a nil descriptor ‚Üí display-only "addresses unavailable" with NO `address.*` call (no wrong-address verify); greps clean of xprv. The shipped single-sig `restoreDocFlow` is NOT reused.
6. **Program nav:** `engraveMultisig` reachable (between engraveSingleSig/qaProgram, non-blank title, no panic, all 3 nav-tests updated), `TestAllocs` green.
7. **Security + no-regression:** typed-only seed (D12 structural + behavioral); per-leg scrub on all exit paths incl. abort/no-match; T4/T5/T6a/single-card flows + codecs byte-unchanged; fuzz `findUserSlot` + the flow (0 panics).

## 6. Invariants (R0 must confirm)
- **I-1 (Critical, D14):** `findUserSlot` matches on the canonical `(chainCode,compressedPubkey)` via `bytes.Equal` over the FULL 32+33 bytes (never base58, never `==` on mismatched array/slice types ‚Äî R0-I2), derives at EACH slot's own origin, REFUSES on zero matches; a non-cosigner seed can NEVER produce an engraved bundle.
- **I-2 (Critical):** the supplied md1 is engraved VERBATIM ‚Äî never re-encoded; the device builds NO multisig md1.

- **I-5:** verify-bundle verifies the USER's slot only (reuse `bundle.Verify` unchanged); other slots unverified-by-design.

- **I-7 (Critical):** seed typed-only (never scan‚Üíderive); ms1 engraved-only/never-NFC; per-leg scrub (incl. the multi-derive cross-match loop) on all exit paths.

## 7. Biggest risks (lock in R0)
1. **D14 cross-match correctness** (I-1) ‚Äî the wrong-wallet guard; canonical-pair compare via `bytes.Equal` over the full 32+33 bytes (R0-I2) + refuse-on-zero + the ambiguous rule; the single net-new logical piece.

4. **Per-leg scrub across the multi-derive cross-match loop** (I-7) ‚Äî scrub on no-match/abort.
5. **No hardware** to validate the supply-gather + cross-match UX.
