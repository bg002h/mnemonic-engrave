## RULINGS ON THE THREE REQUIRED QUESTIONS

### 1. Per-csid `bundleGatherer` reuse soundness — **SOUND.**
The plan wraps `mk1Gatherer`/`md1Gatherer` UNCHANGED in csid-keyed maps (`map[uint32]*mk1Gatherer` / `map[uint32]*md1Gatherer`) and routes each chunk to its csid's sub-gatherer. Verified this is correct against source:

- `mk1Gatherer.offer` (`gui/mk1_inspect.go:55-73`): on the FIRST chunk (`!g.primed`) it primes `total`/`setID` from THAT chunk's header unconditionally (`:60-64`) — it never rejects the priming chunk. The `gatherForeign` arm (`:65-67`) is reached ONLY when already primed AND `h.ChunkSetID != g.setID` (or `!h.Chunked`/`total` mismatch). Since the bundle router creates a NEW sub-gatherer keyed by the chunk's own csid and only ever feeds THAT csid's chunks into it, the `gatherForeign` arm is never hit inside a correctly-routed sub-gatherer. So a foreign csid does NOT break per-csid driving — it simply lands in (or creates) a different map entry. The `md1Gatherer.offer` (`gui/md1_gather.go:30-53`) is structurally identical (priming at `:35-44`, foreign at `:45-47`). `complete()`/`collected()` (`mk1_inspect.go:75-83`, `md1_gather.go:55-63`) support per-csid driving with no hidden global state. **The API genuinely supports being driven this way per-csid.**
- A NEW csid creating a NEW sub-gatherer (plan Task-1 `clsChunkedMK1 → g.mkSets[csid]` "create if new = NEW CARD") IS the correct realization of I-2 (new-csid = new-card). This is the documented inversion vs single-card gather, and it is achieved WITHOUT editing the single-card foreign semantics (I-7 preserved — the plan's Task-1 Step 3 explicitly says "Reuse `mk1Gatherer`/`md1Gatherer` UNCHANGED (do NOT touch their single-card foreign semantics)"). **Correct.**

### 2. `classify` correctness vs the scanner's actual object types — **CORRECT.**
- **ms1 → `codex32.String`** (NOT `mdmkText`): verified `gui/scan.go:70-73` — the scanner tries `codex32.New(string(buf))` (`:70`) BEFORE the `ValidMD/ValidMK → mdmkText` check (`:72-73`). A valid `ms1` parses as a `codex32.String`. The plan's classify (Task-1) has an explicit `codex32.String → clsMs1Refuse` case (R0-C2), and Task-3 surfaces the on-screen refusal — NOT silently dropped. The existing single-card loops type-assert `mdmkText` only (`gui/mk1_inspect.go:218`), confirming the silent-drop defect the fold guards against. The blanket "any `codex32.String` → refuse" is conservatively safe: `codex32.New`'s `inputHRP` (`codex32/checksum.go:77`) is not `ms`-restricted, but the only codex32-shaped strings in this ecosystem are `ms` secrets, and md1/mk1 are caught separately by `ValidMD/ValidMK` — so refusing any `codex32.String` over NFC errs toward the secret-spine intent. **Sound.**
- **md1/mk1 → `mdmkText`**, discriminated by `hasMKPrefix`/`hasMDPrefix`: verified `hasMKPrefix` (`gui/mk1_inspect.go:19`, `mk1`/`MK1`) and `hasMDPrefix` (`gui/md1_inspect.go:15`, `md1`/`MD1`). The HRPs are disjoint and BCH-residue-keyed: `ValidMD` checks HRP `md` (`codex32/mdmk.go:124-125`), `ValidMK` checks HRP `mk` (`:136-143`), case-insensitive (`strings.EqualFold` `:96`). A string valid under one HRP cannot be valid under the other. **The prefix discrimination on a `mdmkText` is reliable.**
- **Single mk1 (`!Chunked`) → REFUSE** (host parity `bundle.rs:128`): verified `mk.ParseHeader` (`mk/mk.go:56`) → `parseHeaderSyms` `typeSingle` arm returns `Header{Chunked:false,...}` (`:74-76`). The plan refuses it (R0-C1) rather than routing into a csid bucket — matching `bundle.rs:128` (`Err(BundleError::Mk1SingleString)`). Justified: `mk.Decode([single])` takes the `!first.Chunked` path returning the fragment with NO cross-chunk hash (`mk/mk.go:177-181`) — an integrity bypass. **Correct.**
- **Single md1 (`!Chunked`) → standalone via `md.Decode`:** verified `md.ParseChunkHeader` single arm returns `ChunkHeader{Chunked:false}`, csid zero-value (`md/chunk.go:193-195`). `md.Decode(s)` (`md/md.go:1216-1230`) does BCH/checksum (`MDDataSymbols` `:1217`), rejects chunked (`:1221-1222`), and full structural decode+validate (`decodePayloadValidated` `:1225`) — this IS the standalone integrity (no chunk set to reassemble). The plan routes singles off the csid map into a non-csid bucket (avoiding the csid-0 collision, R0-C1). **`md.Decode` fully validates a single-string md1 — sound.**
- `mk.ParseHeader`/`md.ParseChunkHeader` reliably yield `Chunked` + csid for routing: confirmed (`mk/mk.go:88` sets `ChunkSetID` for `typeChunked`; `md/chunk.go:197-199` reads the chunk header). **Correct.**
